Authorisation

Basic Authorisation

You can enable Basic Authorisation to allow 3rd party services to access report data. Note however, that this method is insecure and is strongly recommended to keep this option disabled. If you need to allow a 3rd party access to report data, please define an OAuth2 Client.

Basic authorisation status

OAuth2 Clients

Allow and manage 3rd party clients credentials and their access to your data.

By providing 3rd party clients with your randomly generated client identifier and secret, this removes the need to give your own username and password and you can remove a clients access at any time.

NOTE: Always protect the client secret as you would your own password!

Read our OAuth quick start guide for more information and an example of how to use OAuth for secure stats gathering.

Create new client

To create a new client, first provide a name that is meaningful to you.
